gonlang-im

項目背景

  • IM對性能和體驗敏感度高，大廠必備

你將獲得什麼

  • 熟悉開發流程，熟悉技術線 gin+gorm+swagger+log auth等中間件，三高性能

核心功能

  • 發送和接收消息，文字 表情 圖片 語音，訪客，點對點，群聊，快捷回覆，撤回，心跳檢測…

技術線

  • 前端，後端(websocket,channel/goroutine,gin,template,gorm,sql,nosql,mq…)

系統架構

  • 四層：前端，接入層，邏輯層，持久層

系統發送流程

  • A >登陸>權鑑>(遊客)>消息類型>(群/廣播)>B

各個包定義

  1. common->公用
  2. config->環境設定
  3. models->用戶資訊…
  4. router->整理路由
  5. service->業務邏輯
  6. sql->數據庫相關操作
  7. test->測試程式
  8. utils->初始化

1.引入gorm

"gorm.io/driver/mysql" "gorm.io/gorm"

  • 用戶模塊設計
    • 創建modles包user_basic再寫struct
  • 測試數據正確性，然後測試封裝 配制類 測試類
    • 創建test包，編寫test_gorm

2.引入Gin框架

go get github.com/gin-gonic/gin

3.將數據和請求關聯起來

  • 在main方法裡面初始化 配置文件 和 數據庫

utils.InitConfig() utils.InitMysql()

  • 在config包裏面新增app.yaml
  • 新建utils包裏面的system_init.go
  • 到user_basic裏面加上
  • 到service裏面加上userservice
  • 到router包的app.go新增

4. 整合Swagger

go install github.com/swaggo/swag/cmd/swag@latest go get -u github.com/swaggo/gin-swagger go get -u github.com/swaggo/files swag init

  • 測試swagger
    • 127.0.0.1/swagger/index.html ->有跑出來就成功
  • 改善router/app.go 增加完註釋後輸入swag init

5. 日誌打應

  • 修改utils/system_init.go
    • 在數據庫初始化的時候，加入自己的logger

6. 完成用戶模塊基本功能

  • 修改router/app.go

  • 完成基本CRUD

  • 修改user_basic和userservice加入驗證郵箱

  • 修改models/user_basic重複註冊校驗
